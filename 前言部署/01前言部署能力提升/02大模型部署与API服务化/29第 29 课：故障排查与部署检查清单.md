## 故障排查链路

~~~text
浏览器
  ↓
FastAPI：8000 端口
  ↓
参数、鉴权、限流
  ↓
Ollama：11434 端口
  ↓
qwen2.5:3b 模型
~~~

排查时从外到内逐层确认，不要一开始就修改代码。

## 检查 FastAPI 端口

~~~powershell
Get-NetTCPConnection -LocalPort 8000 -State Listen
~~~

作用：检查 FastAPI 是否正在监听 8000 端口。

没有匹配对象通常表示 FastAPI 没有启动，或者服务已经停止。

## 检查 Ollama 端口

~~~powershell
Get-NetTCPConnection -LocalPort 11434 -State Listen
~~~

作用：检查 Ollama 是否正在监听 11434 端口。

## 检查模型

~~~powershell
ollama list
~~~

作用：确认 qwen2.5:3b 是否已经安装。

## 检查 FastAPI 健康接口

~~~powershell
Invoke-RestMethod -Uri "http://127.0.0.1:8000/health"
~~~

正常结果：

~~~json
{
  "status": "ok"
}
~~~

## 实际排查记录

第一次检查 8000 端口时没有找到监听对象。原因不是代码损坏，而是自动化测试使用 TestClient，不需要启动 8000 端口；测试结束后 FastAPI 服务没有运行。

重新执行 start.ps1 后，再次检查得到：

~~~text
127.0.0.1  8000   Listen
127.0.0.1  11434  Listen
~~~

说明 FastAPI 和 Ollama 都已恢复正常。

## 常见提示区分

~~~text
4 passed
~~~

表示自动化测试通过。

~~~text
on_event is deprecated
~~~

表示旧的启动事件写法被标记为过时，不代表本次服务启动失败。

~~~text
favicon.ico 404
~~~

表示浏览器请求图标，但项目没有配置图标，不影响 API。

## 部署检查清单

- Docker 可以运行。
- Ollama 可以运行。
- qwen2.5:3b 模型存在。
- 11434 端口处于 Listen。
- start.ps1 可以启动服务。
- 8000 端口处于 Listen。
- /health 返回 status ok。
- Swagger 页面可以打开。
- 正确 API Key 返回 200。
- 错误 API Key 返回 401。
- 错误请求格式返回 422。
- 超过限流次数返回 429。
- pytest 测试通过。
