# Git/GitHub 基础认知学习记录

## 1. 当前课程定位

本课程属于阶段一：基础建设。

Git/GitHub 是前沿部署学习中的基础工具，主要用于管理部署文档、配置文件、脚本、测试报告、复盘报告和项目作品集。

## 2. Git 是什么

Git 是一个版本控制工具，主要用于记录文件的修改历史。

它可以帮助我们知道：

1. 哪些文件发生了变化。
2. 什么时候发生了变化。
3. 每次变化修改了什么内容。
4. 是否需要回退到以前的版本。

在部署学习和项目交付中，Git 可以管理部署文档、配置文件、脚本和项目代码。

## 3. GitHub 是什么

GitHub 是一个线上代码和文档托管平台。

它可以把本地 Git 仓库上传到线上，方便备份、展示、协作和权限管理。

对于前沿部署学习来说，GitHub 后续可以用于保存：

1. 学习知识库
2. 部署 SOP
3. 环境核验表
4. 脚本文件
5. 项目作品集
6. README 展示页

## 4. Git 和 GitHub 的区别

1. Git 是版本控制工具，GitHub 是线上托管平台。
2. Git 主要在本地记录文件变化，GitHub 用于线上保存、展示和协作。
3. Git 可以单独使用，GitHub 通常配合 Git 使用。
4. Git 关注版本历史，GitHub 还提供仓库管理、协作、Issue、Pull Request、Actions 等功能。

核心关系可以理解为：

```
本地文件夹 -> Git 记录版本 -> GitHub 线上托管和展示
```

## 5. 为什么前沿部署岗位需要 Git/GitHub

前沿部署岗位不仅要安装和配置系统，还要长期维护文档、配置、脚本和交付材料。

使用 Git/GitHub 的原因包括：

1. 部署文档需要版本控制，方便追踪修改历史。
2. 配置文件需要管理，避免误改后无法恢复。
3. 脚本文件需要保存和复用。
4. 测试报告和复盘报告需要归档。
5. 团队成员可能需要协作修改同一份资料。
6. 项目成果可以通过 GitHub 展示给面试官或团队。
7. 出现问题时，可以回看历史版本，定位是哪次修改导致问题。

## 6. Git 核心概念

### 6.1 仓库 Repository

仓库是被 Git 管理的项目文件夹。

例如：

```
前沿部署知识库
Ollama部署项目
RAG-Demo
```

### 6.2 提交 Commit

提交是对某一个时刻文件状态的记录，可以理解为给当前项目拍一张快照。

每次提交都应该写清楚修改内容，例如：

```
新增 Markdown 基础学习记录
补充环境核验 Checklist
更新 Ollama 部署步骤
```

### 6.3 推送 Push

推送是把本地提交上传到 GitHub。

```
本地仓库 -> GitHub 仓库
```

### 6.4 拉取 Pull

拉取是把 GitHub 上的更新同步到本地。

```
GitHub 仓库 -> 本地仓库
```

### 6.5 分支 Branch

分支是在主线之外创建的一条独立修改线。

它可以让不同的人或不同功能并行修改，最后再合并。

### 6.6 合并 Merge

合并是把一个分支上的修改整合到另一个分支中。

例如：

```
把 feature 分支的修改合并到 main 分支
```

GitHub 项目中的 README.md 类似项目说明书，是项目展示入口。

1. 项目目标
2. 项目背景
3. 环境要求
4. 部署步骤
5. 参数配置
6. 使用方法
7. 测试结果
8. 常见问题
9. 截图或运行说明
10. 后续优化方向
