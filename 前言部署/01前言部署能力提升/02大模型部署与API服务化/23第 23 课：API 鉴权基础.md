## 课程目标

第23课在第22课参数验证和错误处理的基础上，为 /chat 接口增加 API Key 鉴权。

本课解决的问题是：

~~~
不是任何人都可以直接调用模型服务
~~~

本课的访问逻辑：

~~~
请求没有 API Key → 401
请求使用错误 API Key → 401
请求使用正确 API Key → 继续调用 Ollama
~~~

## 一、API Key 是什么

API Key 是客户端访问 API 时携带的一段凭证。

服务器收到请求后，会比较：

~~~
客户端提交的 Key
        ↓
服务器保存的 Key
        ↓
两者一致才允许访问
~~~

API Key 的作用类似于一把访问服务的钥匙，但它通常还会和用户身份、权限、额度和有效期绑定。

## 二、本课使用的测试 Key

本课代码中使用：

~~~text
demo-key-123
~~~

这个 Key 是我们手动设定的学习用测试值，不是系统自动生成的正式密钥，也不能用于生产环境。

正式平台的 API Key 通常由后台使用安全随机算法生成，再交给用户使用。

~~~python
from fastapi import FastAPI, HTTPException, Header
from pydantic import BaseModel, Field
import requests

app = FastAPI()

OLLAMA_URL = "http://localhost:11434/api/generate"
MODEL = "qwen2.5:3b"
API_KEY = "demo-key-123"


class ChatRequest(BaseModel):
    message: str = Field(
        ...,
        min_length=1,
        max_length=2000
    )


@app.get("/health")
def health():
    return {"status": "ok"}


@app.post("/chat")
def chat(
    request: ChatRequest,
    x_api_key: str | None = Header(default=None)
):
    if x_api_key != API_KEY:
        raise HTTPException(
            status_code=401,
            detail="API Key 无效或缺失"
        )

    payload = {
        "model": MODEL,
        "prompt": request.message,
        "stream": False
    }

    try:
        response = requests.post(
            OLLAMA_URL,
            json=payload,
            timeout=120
        )

        response.raise_for_status()
        data = response.json()

        if not data.get("response"):
            raise HTTPException(
                status_code=502,
                detail="Ollama 返回内容为空"
            )

        return {
            "reply": data["response"],
            "model": data.get("model"),
            "done": data.get("done")
        }

    except requests.exceptions.Timeout:
        raise HTTPException(
            status_code=504,
            detail="Ollama 响应超时"
        )

    except requests.exceptions.ConnectionError:
        raise HTTPException(
            status_code=502,
            detail="无法连接 Ollama"
        )

    except requests.exceptions.RequestException as error:
        raise HTTPException(
            status_code=502,
            detail=f"Ollama API 调用失败：{error}"
        )
~~~

保存并关闭记事本。

## 四、鉴权代码含义

### 1. 导入 Header

~~~python
from fastapi import FastAPI, HTTPException, Header
~~~

Header 用来读取 HTTP 请求头中的内容。

### 2. 设置服务器端 Key

~~~python
API_KEY = "demo-key-123"
~~~

这表示服务器暂时保存一个固定的测试 Key。

### 3. 接收客户端 Key

~~~python
x_api_key: str | None = Header(default=None)
~~~

FastAPI 会从请求头中读取名为：

~~~text
x-api-key
~~~

的字段。

代码中的变量名是 x_api_key，FastAPI 默认会把下划线转换为请求头中的短横线，因此 Swagger 页面显示的是 x-api-key。

### 4. 比较 Key

~~~python
if x_api_key != API_KEY:
~~~

比较客户端提交的 Key 和服务器保存的 Key。如果不一致，就拒绝访问。

### 5. 返回 401

~~~python
raise HTTPException(
    status_code=401,
    detail="API Key 无效或缺失"
)
~~~

401 Unauthorized 表示请求没有通过身份验证。此时请求不会继续调用 Ollama。

## 七、遇到 401 时如何排查

如果输入正确 Key 却返回 401，先检查：

1. Swagger 字段名称是否为 x-api-key。
2. 是否填写了 demo-key-123。
3. 是否误把 Key 写进了 JSON 请求体。
4. Key 前后是否有空格或多余引号。
5. Swagger 生成的 Curl 中是否包含：

~~~text
-H 'x-api-key: demo-key-123'
~~~

如果 Curl 中没有这一行，说明请求头没有真正发送出去。

## 八、API Key 是怎么生成的

当前代码中的：

~~~python
API_KEY = "demo-key-123"
~~~

是我们手动设定的测试 Key。

正式系统通常使用安全随机算法生成，例如：

~~~powershell
python -c "import secrets; print(secrets.token_urlsafe(32))"
~~~

secrets.token_urlsafe(32) 会生成一串适合用于凭证的随机字符串。

正式平台申请 API Key 时，平台后台会负责：

- 生成随机 Key
- 将 Key 和用户账号绑定
- 设置权限和有效期
- 保存或验证 Key
- 支持撤销和重新生成

## 九、为什么 Key 不应该直接写在代码里

代码中的固定 Key 适合课堂演示，但不适合生产环境，因为：

- 代码可能被上传到 GitHub。
- 多人共享代码时容易泄露。
- 修改 Key 需要修改代码并重新部署。
- 不方便区分不同用户。

更安全的做法是使用环境变量：

~~~python
import os

API_KEY = os.getenv("API_KEY")
~~~

然后在 PowerShell 中设置：

~~~powershell
$env:API_KEY = "你的随机密钥"
~~~

这样 Key 不会直接写进 main.py。
