## 课程定位阶段二：核心技能
模块：大模型部署深化 / API 服务化基础
上一课：完成 Ollama API 调用
本课：看懂 API 返回结果

## 目标
1. API 返回结果里每个字段大概是什么意思
2. 哪些字段用于判断调用是否成功
3. 哪些字段用于判断性能和耗时
4. 为什么部署人员要看返回结果，而不是只看有没有回答

## 一、为什么要学返回字段
部署岗位不能只是看模型有回答，还需要看：
请求是否完成
用的是哪个模型
耗时是多少
生成了多少 token
模型加载慢不慢

## 二、第一组：调用结果字段
model：本次实际调用的模型名称。比如（qwen2.5:3b）
response：模型返回的正文内容。（这是业务上最关心的字段）
done：是否生成完成。done=True 结束原因。done=True（说明这次请求正常结束）
done_reason（stop表示模型正常停止）

## 三、第二组：时间性能字段
total_duration：本次请求总耗时。它包括模型加载、理解问题、生成回答等过程
load_duration：模型加载耗时。如果这个很大，说明模型从硬盘加载到内存/显存花了较久
prompt_eval_duration：模型处理输入问题的耗时。prompt 很长，这个可能变大。
eval_duration：模型生成回答的耗时。回答很长，或者模型很大，这个可能变大
（用于判断慢在哪里）
注意：Ollama 返回的 duration 通常是纳秒，数字会很大。你先不用算得很细，只要知道谁大谁小
load_duration 看加载慢不慢。
prompt_eval_duration 看理解问题慢不慢。
eval_duration 看生成回答慢不慢。
total_duration 看整体慢不慢。

## 四、第三组：token 数量字段
prompt_eval_count：输入内容被模型处理了多少 token。
eval_count：模型生成了多少 token。
token：模型处理文本的基本单位
输入越长，prompt_eval_count 越大。
回答越长，eval_count 越大。

## 五、最重要的排查思路
API调用如果很慢的情况
load_duration 很大：模型加载慢
prompt_eval_duration 很大：输入内容太长或处理慢
eval_duration 很大：生成回答慢
eval_count 很大：回答内容太长
通过这些字段去排查

实际判断 API 成功，最常看：
技术完成看 done=True 和 done_reason=stop。
是否真的有回答，要看 response 是否有内容。

如果回答特别长，哪个 count 字段会变大？
eval_count = 模型生成了多少 token
eval_duration = 模型生成这些 token 花了多长时间
count 看数量
duration 看时间
成功判断：response / done / done_reason
性能判断：total_duration / load_duration / eval_duration / eval_count
