# Obsidian 知识库搭建记录

## 1. 当前课程定位

本课程属于阶段一：基础建设。

Obsidian 的作用不是单纯记录零散笔记，而是建立一个能够长期沉淀学习内容、部署项目、交付文档和面试材料的个人前沿部署知识库。

## 2. 对 Obsidian 的理解

Obsidian 是一个本地化知识库工具，底层使用 Markdown 文件保存内容。它可以通过双向链接、标签和文件夹结构，把不同笔记连接起来，让学习内容、部署流程和项目文档形成体系。

在前沿部署学习中，Obsidian 可以作为：

1. 个人部署知识库
2. 项目交付资料库
3. 问题排查经验库
4. 面试作品素材库

## 3. 为什么学习前沿部署需要 Obsidian

前沿部署岗位需要学习多个方向，包括系统部署、AI 大模型部署、GitHub、ERP/WMS、技术文档和 SOP。如果只零散记录，很难形成完整能力。

使用 Obsidian 的原因包括：

1. 可以把多个技术模块的笔记串联起来。
2. 可以用 Markdown 统一管理学习笔记和部署文档。
3. 可以通过双向链接连接概念、流程、模板和项目。
4. 可以沉淀问题记录和解决方案，方便后续复盘。
5. 可以逐步整理成面试作品集和交付材料。

## 6. 需要记录的笔记类型

### 6.1 概念笔记

用于解释核心概念，例如：

- 什么是端口
- 什么是 API
- 什么是 Token
- 什么是 Embedding

### 6.2 流程笔记

用于记录完整流程，例如：

- 系统部署完整流程
- 环境核验流程
- Ollama 部署流程

### 6.3 部署笔记

用于记录具体部署步骤，例如：

- Ollama 本地部署 SOP
- Open WebUI 部署 SOP
- Odoo ERP 部署记录

### 6.4 项目笔记

用于记录真实项目成果，例如：

- 项目目标
- 环境信息
- 部署步骤
- 测试结果
- 问题和解决方案
- 项目复盘

### 6.5 问题反馈笔记

用于记录报错、原因和处理方式，例如：

- 端口被占用
- 模型下载失败
- 网页打不开
- 权限不足

## 7. 后续插件准备

第一阶段先不急于安装大量插件，优先掌握 Markdown、文件夹结构、双向链接和标签。

后续建议插件：

1. Calendar：用于学习日记和每周复盘。
2. Templater：用于快速生成学习笔记、部署 SOP 和复盘模板。
3. Dataview：用于查询项目、任务、标签和交付文档。
4. Git：用于把 Obsidian 知识库同步到 GitHub。

插件使用原则：

```
先建立内容体系，再安装插件增强效率。
插件服务于学习交付，不为了复杂而复杂。
```
