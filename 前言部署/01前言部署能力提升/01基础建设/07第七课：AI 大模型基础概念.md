## 一、LLM 是什么
**LLM 全称是 Large Language Model，大语言模型**
能理解和生成自然语言的 AI 模型
例如：ChatGPT，Claude，通义千问，文心一言，Llama，Qwen，DeepSeek
在部署岗位需要知道
模型文件怎么运行
需要多少内存/显存
怎么通过工具调用
怎么变成网页或 API 服务

## 二、Token 是什么
**Token 可以理解成模型处理文本的基本单**
模型不是按整篇文章理解，而是把文本切成很多 token 来处理。
输入和输出越长消耗的Token越多

## 三、Context Window 是什么
**Context Window 叫上下文窗口**
它表示模型一次能“记住”和处理多少内容
上下文越大，模型一次能处理的内容越多，但也更消耗显存和计算资源
**上下文越大，资源压力越大**
机器配置不足时，长文本容易变慢或失败

## 四、Embedding 是什么
**Embedding 叫向量化**
把文字变成一串数字，让计算机能比较文字之间的相似度。
**它是 RAG 文档问答的基础**
提供问题-问题变成向量-再去文档搜索-大模型回答

**Embedding 是 RAG 检索的基础。**

## 五、模型参数和模型大小
7B，14B，32B，70B
B是 Billion，十亿。
参数越大，能力可能越强
参数越大，占用资源也越高
**本地部署部署越大越好，需要看电脑配置能不能跑的动**
如果机器配置一般，优先选择 7B 量化模型。
如果有高显存显卡，可以尝试更大的模型。
如果只是做演示，先跑通比追求大模型更重要。

## 六、本地部署是什么意思
本地部署就是模型运行在自己的电脑或公司服务器上，而不是完全依赖外部平台
优点：数据更可控，可以内网使用，适合私有化场景，可以定制服务

缺点：需要硬件资源，需要自己维护，部署和排查成本更高，模型效果受硬件限制
后面部署：Ollama + 本地模型
进一步：Ollama + Open WebUI
成品：本地模型服务 + 网页访问界面

## 岗位表达
大模型部署不是只下载模型，而是根据业务需求和硬件环境，选择合适的模型和部署工具，让模型能够稳定运行，并通过命令行、网页或 API 方式提供服务。部署时需要关注模型大小、内存、显存、上下文长度、端口、网络和后续运维

# AI 大模型基础概念学习记录

## 1. 当前课程定位

本课程属于阶段一：基础建设。

本课目标是为后续 Ollama 本地部署做准备，先理解大模型部署中常见的基础概念，包括 LLM、Token、Context Window、Embedding、模型参数和本地部署。

## 2. LLM 是什么

LLM 是 Large Language Model 的缩写，中文是大语言模型。

它是一类能够理解和生成自然语言的 AI 模型，可以用于对话、问答、总结、翻译、代码辅助、文档生成等任务。

常见大语言模型包括：

```
ChatGPT
Claude
通义千问
文心一言
Llama
Qwen
DeepSeek
```

在前沿部署岗位中，重点不是研究模型如何训练出来，而是理解模型如何运行、如何部署、需要多少硬件资源、如何通过网页或 API 提供服务。

## 3. Token 是什么

Token 是大模型处理文本时使用的基本单位。

它不完全等于一个字或一个词，可以简单理解为模型内部处理文本时拆分出来的小片段。

部署时需要关注 Token，因为：

1. 输入越长，Token 越多。
2. 输出越长，Token 越多。
3. Token 越多，计算量和资源消耗越大。
4. 长文本处理会增加内存、显存和响应时间压力。

## 4. Context Window 是什么

Context Window 是上下文窗口，表示模型一次能够处理和记住的文本范围。

常见表达包括：

```
4K 上下文
8K 上下文
32K 上下文
128K 上下文
```

上下文窗口越大，模型一次能处理的内容越多，长文本能力越强。

但上下文越大，也会带来更高的硬件资源消耗，尤其会增加显存和计算压力。

因此部署时需要根据业务场景和硬件条件选择合适的上下文长度。

## 5. Embedding 是什么

Embedding 可以理解为向量化。

它的作用是把自然语言文本转换成一组数字向量，让系统可以计算文本之间的相似度。

在 RAG 文档问答中，Embedding 的作用非常关键：

1. 将用户问题转换成向量。
2. 将知识库文档也转换成向量。
3. 通过相似度检索找到最相关的文档片段。
4. 再把检索到的内容交给大模型生成回答。

所以，Embedding 是 RAG 检索环节的基础。

注意：Embedding 不是让 AI “直接理解全部文档”，而是帮助系统找到和问题最相关的资料。

## 6. 7B、14B、32B 是什么意思

7B、14B、32B 中的 B 是 Billion，表示十亿。

例如：

```
7B = 70 亿参数
14B = 140 亿参数
32B = 320 亿参数
```

一般来说，参数规模越大，模型能力可能越强，但也需要更多硬件资源。

部署时需要关注：

1. 模型越大，占用磁盘空间越多。
2. 模型越大，运行时内存和显存要求越高。
3. 模型越大，推理速度可能越慢。
4. 普通电脑通常更适合先运行 7B 或量化模型。

参数大小和模型能力通常相关，但不能简单理解为“参数越大知识越多就一定越好”。模型效果还受训练数据、模型架构、量化方式和部署环境影响。

## 7. 什么是本地大模型部署

本地大模型部署，是指将大语言模型运行在个人电脑、公司服务器或内网环境中，而不是完全依赖外部云端平台。

优点：

1. 数据更可控。
2. 可以在内网或私有化环境中使用。
3. 更适合对数据安全有要求的场景。
4. 可以根据业务需求进行服务封装和定制。

缺点：

1. 需要硬件资源。
2. 需要人员维护。
3. 部署和问题排查成本较高。
4. 模型效果和运行速度受硬件配置限制。

## 8. 部署、微调和训练的区别

这是本课最重要的纠正点。

大模型部署不等于训练大模型。

三者区别如下：

|概念|含义|难度|
|---|---|---|
|部署|把已有模型运行起来，并提供命令行、网页或 API 服务|入门到中级|
|微调|用特定领域数据对已有模型进行再训练，让模型更适合某类任务|中高级|
|训练|从大量数据和算力开始训练一个模型|高级|

前沿部署助理岗位的入门阶段，重点是先会部署已有模型，例如使用 Ollama 运行 Qwen、Llama、DeepSeek 等开源模型。
