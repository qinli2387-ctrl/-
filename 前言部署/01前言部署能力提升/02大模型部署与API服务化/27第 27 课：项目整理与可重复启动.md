## 本课学什么

本课学习如何把启动过程固定下来，避免每次都手动输入多条命令。

## 通过什么实现

~~~text
requirements.txt
    ↓
记录项目依赖

环境变量
    ↓
提供模型、地址和 API Key

start.ps1
    ↓
统一设置配置并启动服务
~~~

## 创建启动脚本

~~~powershell
notepad start.ps1
~~~

完整脚本：

~~~powershell
$ErrorActionPreference = "Stop"

Set-Location $PSScriptRoot

$env:MODEL = "qwen2.5:3b"
$env:OLLAMA_URL = "http://localhost:11434/api/generate"

if ([string]::IsNullOrWhiteSpace($env:API_KEY)) {
    $env:API_KEY = Read-Host "Enter API Key"
}

if ([string]::IsNullOrWhiteSpace($env:API_KEY)) {
    Write-Error "API_KEY is empty"
    exit 1
}

Write-Host "Project: $PSScriptRoot"
Write-Host "Model: $env:MODEL"
Write-Host "Ollama: $env:OLLAMA_URL"
Write-Host "API key loaded"

python -m uvicorn main:app --reload
~~~

## 启动命令

~~~powershell
powershell -ExecutionPolicy Bypass -File .\start.ps1
~~~

这条命令会执行启动脚本。脚本会自动定位项目目录、设置模型和 Ollama 地址，并加载 API Key。

## 遇到的问题

最初脚本中的中文输出导致 PowerShell 解析字符串时报错：

~~~text
TerminatorExpectedAtEndOfString
~~~

后来将启动脚本中的提示文字改为 ASCII 英文，脚本成功运行。

## 验证结果

启动后看到：

~~~text
Project: C:\Users\Administrator\Desktop\fastapi-demo
Model: qwen2.5:3b
Ollama: http://localhost:11434/api/generate
API key loaded
Application startup complete.
~~~

说明项目目录、模型、Ollama 地址、API Key 和 FastAPI 服务都已加载成功。

## 本课结论

start.ps1 是启动流程的自动化入口。它不负责业务逻辑，只负责准备环境并启动 main.py。
