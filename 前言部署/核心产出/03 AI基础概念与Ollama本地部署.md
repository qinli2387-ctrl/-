## 一，目标
使用 Ollama 在 Windows 本地运行第一个开源大模型
## 二、AI基础概念
- LLM：能够理解和生成自然语言的大语言模型。
    
- Token：模型处理文本时使用的基本单位，输入和输出越长，消耗越多。
    
- Context Window：模型一次能够处理的上下文范围。
    
- Embedding：把文本转换成向量，是后续 RAG 检索的基础。
    
- 7B、14B、32B：表示模型参数规模，模型越大，通常需要更多内存和显存。
    
- 本地部署：把已有模型运行在自己的电脑或公司服务器上。
    
- 部署、微调、训练是不同概念，本阶段重点是部署已有模型。
## 三、Ollama 的作用
Ollama 是本地大模型运行和管理工具，不是大模型本身
他负责能做的内容
1. 下载模型。
    
2. 管理模型文件。
    
3. 启动模型服务。
    
4. 接收用户输入。
    
5. 返回模型回答。
## 四、本地部署流程
A[检查系统和硬件环境] --> B[安装 Ollama]
B --> C[检查 Ollama 版本]
C --> D[下载并运行模型]
D --> E[发送测试问题]
E --> F[确认模型返回回答]
F --> G[正常退出或继续提供服务]
## 五、实际操作流程
### 1. 检查 Ollama
ollama --version （作用：确认 Ollama 已安装，并且命令行可以识别）
### 2. 运行第一个模型
ollama run qwen3:0.6b （作用：下载并运行 `qwen3:0.6b` 模型）
### 3. 进行中文问答测试
提问让他进行回答，能知道他在运行了
### 4. 退出模型
/bye
## 六、部署成功标准
1. `ollama --version` 能显示版本。
    
2. 模型能够成功下载。
    
3. 模型能够进入对话状态。
    
4. 中文问题能够得到回答。
    
5. 可以正常退出模型。
    
6. 后续能够通过网页或 API 调用模型。
