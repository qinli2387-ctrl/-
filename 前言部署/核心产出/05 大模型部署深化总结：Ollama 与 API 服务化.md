## 一、学习目标
在本地运行大语言模型，并通过 FastAPI 将模型封装成其他程序可以调用的 API 服务
## 二、整体架构
```
A[调用方或网页] --> B[FastAPI API 服务]
B --> C[API Key 鉴权]
C --> D[请求参数验证]
D --> E[请求限流]
E --> F[Ollama API]
F --> G[qwen2.5:3b 模型]
G --> H[返回模型结果]
H --> B
B --> A
```
## 三、各组件作用
- Ollama：负责在本地运行和管理大模型。
    
- qwen2.5:3b：实际提供回答能力的模型。
    
- FastAPI：接收外部请求，并把请求转发给 Ollama。
    
- API Key：验证调用者是否有权限。
    
- 参数验证：检查请求内容是否符合要求。
    
- 请求限流：防止短时间内请求过多。
    
- 日志和指标：记录服务运行状态，方便排查问题。
    
- Swagger：通过网页测试和验证 API。
## 四、实际服务配置
```
FastAPI 地址：http://127.0.0.1:8000
Ollama 地址：http://localhost:11434/api/generate
模型名称：qwen2.5:3b
```
## 五、已完成的接口
### 1. 健康检查
GET /health （作用：确认 FastAPI 服务是否正常运行）
### 2. 对话接口
POST /chat （作用：接收用户问题，调用 Ollama 模型，并返回模型回答）
请求体：
{
  "message": "请解释什么是系统部署"
}
返回体：
```
{
  "reply": "模型回答内容",
  "model": "qwen2.5:3b",
  "done": true
}
```
### 3. 运行指标接口
GET /metrics （作用：查看请求数量、成功数量、失败数量等运行数据）

## 八、Ollama 与 vLLM 的关系
Ollama 适合本地快速部署和测试模型，配置简单，适合当前学习阶段。

vLLM 更适合 GPU 服务器和生产环境，重点提升模型推理吞吐量和并发能力。

本阶段已经完成 Ollama + FastAPI 实操，vLLM 作为后续高性能模型服务方向继续学习
