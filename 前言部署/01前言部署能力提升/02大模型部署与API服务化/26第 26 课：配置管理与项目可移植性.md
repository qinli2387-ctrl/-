## 课程目标

第26课学习如何把项目配置从 Python 代码中分离出来，让项目更容易迁移和重新部署。

本课主要解决：

- API Key 不直接写在代码中。
- 模型名称可以通过环境变量修改。
- Ollama 地址可以通过环境变量修改。
- 依赖可以通过 requirements.txt 重新安装。

## 一、配置和代码的区别

原来的写法是把配置写死在代码里：

~~~python
OLLAMA_URL = "http://localhost:11434/api/generate"
MODEL = "qwen2.5:3b"
API_KEY = "demo-key-123"
~~~

这种方式虽然能运行，但不方便部署。

本课改成：

~~~text
PowerShell 环境变量
        ↓
Python 使用 os.getenv() 读取
        ↓
FastAPI 使用配置运行
~~~

代码负责业务逻辑，环境变量负责运行配置。

## 二、依赖清单 requirements.txt

创建文件：

~~~powershell
notepad requirements.txt
~~~

写入：

~~~text
fastapi==0.139.0
uvicorn==0.51.0
requests==2.34.2
~~~

安装依赖：

~~~powershell
python -m pip install -r requirements.txt
~~~

### 这些依赖的作用

- fastapi：编写 API 服务。
- uvicorn：启动 FastAPI 服务。
- requests：调用 Ollama HTTP API。

requirements.txt 的意义是把当前项目需要的依赖记录下来。以后换电脑时，只需要执行一次安装命令，不需要手动猜测应该安装哪些包。

## 三、检查依赖

~~~powershell
python -m pip freeze | Select-String "fastapi|uvicorn|requests"
~~~

实际检查结果：

~~~text
fastapi==0.139.0
requests==2.34.2
uvicorn==0.51.0
~~~

说明当前 Python 环境已经满足项目运行要求。

## 四、设置运行配置

进入项目目录：

~~~powershell
cd "$env:USERPROFILE\Desktop\fastapi-demo"
~~~

设置模型名称：

~~~powershell
$env:MODEL = "qwen2.5:3b"
~~~

设置 Ollama 地址：

~~~powershell
$env:OLLAMA_URL = "http://localhost:11434/api/generate"
~~~

生成随机 API Key：

~~~powershell
$env:API_KEY = python -c "import secrets; print(secrets.token_urlsafe(32))"
~~~

查看当前 Key：

~~~powershell
$env:API_KEY
~~~

API Key 应该只复制到本地 Swagger 的 x-api-key 输入框，不要发到聊天或公开代码仓库。

## 五、Python 读取环境变量

~~~python
import os

OLLAMA_URL = os.getenv("OLLAMA_URL")
MODEL = os.getenv("MODEL")
API_KEY = os.getenv("API_KEY")
~~~

os.getenv 的作用是从当前进程的环境变量中读取配置。

如果没有设置配置，可以主动报错：

~~~python
if not OLLAMA_URL:
    raise RuntimeError("缺少 OLLAMA_URL 环境变量")

if not MODEL:
    raise RuntimeError("缺少 MODEL 环境变量")

if not API_KEY:
    raise RuntimeError("缺少 API_KEY 环境变量")
~~~

这样配置缺失时，服务会在启动阶段直接说明原因，而不是运行到请求时才出现难以理解的错误。

## 六、启动服务

~~~powershell
python -m uvicorn main:app --reload
~~~

启动时终端会显示类似：

~~~text
当前模型：qwen2.5:3b
Ollama 地址：http://localhost:11434/api/generate
Application startup complete.
~~~

日志中显示模型和地址，但不显示 API Key，这样可以避免密钥进入日志。

## 七、验证完整链路

打开：

~~~text
http://127.0.0.1:8000/docs
~~~

在 POST /chat 中填写刚才生成的随机 Key，发送：

~~~json
{
  "message": "请用一句话解释什么是配置管理。"
}
~~~

实际验证结果包含：

~~~json
{
  "reply": "配置管理是通过自动化手段持续记录、变更和分发系统及其组件的配置信息的过程。",
  "model": "qwen2.5:3b",
  "done": true
}
~~~

## 八、验证结果含义

- 200：FastAPI 成功处理请求。
- reply：Ollama 返回的模型回答。
- model：配置中的模型名称被成功读取并使用。
- done 为 true：模型生成已经完成。
- 能够返回回答：OLLAMA_URL 配置正确，且 Ollama 11434 端口可以访问。
- 鉴权通过：API_KEY 环境变量和 Swagger 提交的 Key 一致。

## 九、环境变量的生命周期

使用 $env: 的方式设置的变量只对当前 PowerShell 窗口有效：

~~~text
当前窗口存在 → FastAPI 可以读取
关闭窗口 → 环境变量消失
~~~

因此重新打开窗口后，需要重新设置配置。后续可以学习 .env 文件、Windows 持久环境变量或密钥管理工具。

## 十、本课的完整配置链路

~~~text
requirements.txt
    ↓
安装 FastAPI、Uvicorn、requests
    ↓
PowerShell 设置 MODEL、OLLAMA_URL、API_KEY
    ↓
Python 使用 os.getenv() 读取
    ↓
FastAPI 启动并使用配置
    ↓
调用 Ollama
    ↓
返回模型回答
~~~
